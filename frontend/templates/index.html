What it does

A structured diagnostic report from a single image.

The screening pipeline accepts any clear nail photograph. It runs the image through a pretrained VGG-16 classifier, returns the top prediction with a calibrated confidence score, maps it to a curated disease record, and delivers a complete report — description, observable symptoms, severity grade, and next steps — in one API response.

Primary prediction The most probable condition with a confidence percentage
Top-5 alternatives Ranked differential with probabilities for each candidate
Severity grade Low / Moderate / High classification per condition
Symptom mapping Curated list of observable clinical signs
Clinical guidance Recommended next steps appropriate to severity
How it works

Four stages, under one second.

01

Upload & validate

The image passes extension whitelist, MIME-type verification, and size enforcement before any processing begins. Malformed files are rejected at the boundary.

02

Preprocess

PIL converts to RGB, Lanczos resampling resizes to 224×224, pixel values are normalized to [0, 1], and the tensor is shaped for batch inference.

03

Inference

VGG-16 performs a forward pass and returns softmax probability distributions across all 17 pathology classes simultaneously.

04

Report

Top probabilities are mapped to enriched disease records. The complete response — prediction, scores, severity, symptoms, guidance — is returned as structured JSON.

Architecture

The stack, without abstraction.

Model
VGG-16 — Oxford's 16-layer convolutional network, transfer-learned with frozen early layers and a retrained classification head for nail pathology. Softmax output, 17 classes.
Framework
TensorFlow / Keras — Model loading, tensor preprocessing, and synchronous inference. Production-stable with a predictable memory footprint.
Backend
Flask — REST endpoints with structured JSON responses. Standard multipart POST. Integrates directly into EMR pipelines or mobile clients.
Processing
Pillow + NumPy — Image decode, RGB conversion, Lanczos resampling to 224×224, float32 normalization, and batch dimension expansion.
