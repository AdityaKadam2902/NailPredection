Architecture

The technology behind NailCareAI.

Transfer learning on proven convolutional architectures, validated preprocessing, and a structured inference pipeline purpose-built for nail pathology.

Deep Learning

Transfer learning models.

All models use ImageNet pretrained weights with domain-specific fine-tuning on curated nail disease imagery.

VGG-16 Primary Model

16 layers · 138M parameters · 224×224 input

Our production model. Oxford's VGG-16 architecture — 13 convolutional layers with 3 fully connected heads. We froze the convolutional stack and retrained the dense layers on the nail disease dataset. This preserves low-level feature extraction quality from ImageNet while adapting the classifier to our 17-class distribution. Softmax output produces calibrated per-class probabilities.

ResNet-50 Residual Network

50 layers · Skip connections · 25M parameters

Skip connections allow gradient flow through very deep networks without degradation. ResNet-50 provides strong hierarchical feature extraction and serves as a comparison baseline in experimental settings.

Skip Connections Deep Architecture Baseline Comparison
InceptionV3 Multi-Scale

48 layers · Parallel filters · 23M parameters

Inception modules apply 1×1, 3×3, and 5×5 convolutions in parallel, capturing texture and morphology at different spatial scales simultaneously — particularly suited to nail surface analysis where both fine grain and shape matter.

Parallel Filters Multi-Scale Features
Xception Efficient

36 layers · Depthwise separable · 22M parameters

Depthwise separable convolutions decouple channel-wise and spatial filtering, drastically reducing parameter count while maintaining representational capacity. Evaluated for deployment scenarios where inference latency is constrained.

Pipeline

Inference pipeline, step by step.

01

Secure Upload

Incoming files pass a four-layer validation chain: magic number check (file signature bytes), extension whitelist, MIME-type verification, and maximum size enforcement. Malformed or oversized files are rejected before touching model infrastructure.

02

Preprocessing

PIL decodes the image and forces conversion to RGB (handles grayscale, RGBA, CMYK inputs). Lanczos resampling resizes to 224×224. NumPy converts to float32 and normalizes pixel values to the [0, 1] range. The tensor is expanded to a batch dimension of 1 before inference.

03

Model Inference

The preprocessed tensor is fed to the loaded VGG-16 model via model.predict(). The model returns a (1, 17) softmax probability array. Top-5 indices are extracted by probability rank and mapped to class label strings from the training vocabulary.

04

Result Enrichment

The primary class label is used to look up a structured disease record containing medical description, symptom list, severity grade (Low / Moderate / High), and recommended next steps. This enriched response is serialized to JSON and returned to the client.
